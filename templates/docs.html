Organization

Each organization is represented as an entity and has:

  • n number of users.
  • n number of projects.
  • a single payment entity, can be managed by multiple persons inside the organization.

User Types

There are two types of users: Contributors and Collaborators.

Contributors

Contributors have the following permissions and responsibilities:

  • Access and manage projects and segments.
  • Invite and manage users within the organization.
  • Approve and oversee tasks and milestones.
  • Access to all levels of the project, including ticket and post levels.
  • Can perform bulk updates and manage user roles within the system.

Collaborators

Collaborators have the following permissions and responsibilities:

  • Access to tickets and posts within assigned segments.
  • Manage and update tickets and posts.
  • View project milestones and tasks but with limited edit permissions.
  • Can contribute to discussions and provide updates on tickets and posts.

Registration and User Management

The first user to register for an organization will have to invite the remaining users by allowing them access to specific segments or the entire organization sub-domain. This user can also assign the contributor user role to invited users.

Important Points

  • If an organization already exists, new users need to contact their contributor for access.
  • Payment is managed at the organization level, with a single payment account sufficient for all users within an organization.
  • Any root user or contributor within the organization can handle payments.
  • Deleted organizations can be reactivated within 3 months, after which new registrations may be required under a different organization name.

Soft Deletion and Data Retention

Organizations and user accounts that are marked for deletion will remain in the system for a period of 3 months. During this period, if a user wants to reactivate their account or organization, they can contact us for manual restoration.

Projects

Projects are the main initiatives that need to be managed and tracked within the system. They serve as containers for various phases, tasks, and assignments, allowing for organized and efficient project management. Each project has a

  • unique title.
  • description.
  • start and end dates.
  • status (open or closed).
  • associated members and phases.

Segments

Segments are sub-parts or sub-divisions or sub-projects within projects that represent distinct phases or parts of the project's lifecycle. They allow for better organization and management of tasks, ensuring that each segment is handled by the appropriate team members.

  • unique title.
  • description.
  • start and end dates.
  • status (open or closed).
  • associated members and assignments.
  • and mostly one or more stages.

Tickets

Tickets represent a task or an issue within a project management system. They are used to track progress, assign responsibilities, and manage the details related to individual tasks or issues.

  • A ticket can be in a single segment at a single time.
  • A ticket is associated with stages.
  • A ticket can be in a single stage at a single time.
  • A ticket can be moved to other stages or segments as per the requirement.
  • Each ticket will have a unique number followed by # key. For example ticket: #11
  • Each ticket has root tickets and branch tickets.
  • Root tickets are parent tickets.
  • Branch tickets are child tickets.
  • No need to complete all tickets in order to complete the task.
  • Each ticket can have zero or more tags.

Posts

Posts are the solutions / resolutions / thoughts posted by the user for a single issue / ticket.

  • A ticket can have zero or more posts.
  • Any user assigned to a segment or ticket can post on the ticket page.
  • A post can have n number of upvotes and n number of downvotes given by all users.
  • The user who created the ticket can accept any one of the posts as an answer or a solution.
  • Users can reply to posts.
  • Users can pin posts. Other users will see that it is a pinned post.
  • Users can save posts. Later they can only see all the saved posts.

Milestones

A Milestone is a significant checkpoint or achievement reached. It helps track progress and manage deadlines by defining key objectives or deliverables.

Sprints

A sprint is a set time period within which specific tasks or work items are completed and reviewed in agile project management methodologies, particularly in Scrum. It represents a fixed-duration iteration of work aimed at achieving predefined goals or delivering incremental value to the project.

  • Duration:

    Description: Typically, a sprint lasts for 2-4 weeks, but this can vary based on the project's needs. It's a recurring cycle where teams focus on completing a set of tasks or user stories.

  • Goals:

    Description: Each sprint has specific objectives or goals that are defined during sprint planning. These goals guide the team's work and are aligned with the overall project roadmap.

  • Meetings:

    Description: Sprint meetings typically include sprint planning, daily stand-ups, sprint reviews, and retrospectives. These meetings help ensure that the team stays on track and can address any issues that arise.

  • Deliverables:

    Description: At the end of each sprint, the team is expected to deliver a potentially shippable product increment or completed work items that meet the defined goals.
